RL1

Deterministic Finite Automata (DFA)

🌍 What Is This All About?

Imagine you need to teach a computer to recognise valid email addresses, or to check whether someone typed a correct phone number. The computer needs a model — a precise description of "what counts as valid". That model is called a formal language, and the machine that checks it is called a finite automaton.

The key insight of this whole unit is: the same collection of strings can be described in at least three equivalent ways — by a DFA, by an NFA, or by a regular expression. All three give you regular languages.

📖 Alphabets, Strings, and Languages ⭐EXAM

Definition
  • Alphabet (Σ) — a finite, non-empty set of symbols, e.g. Σ = {0, 1} or Σ = {a, b, c}.
  • String (word) — a finite sequence of symbols from Σ. The empty string is written ε (length 0).
  • |w| — the length of string w. Example: |aba| = 3, |ε| = 0.
  • Σ* — the set of all finite strings over Σ, including ε.
  • Language L — any subset of Σ*. A language is simply a collection of strings we decide to call "valid".
Plain English Example

Σ = {0, 1}. Then Σ* = {ε, 0, 1, 00, 01, 10, 11, 000, …} — infinitely many strings.

The language "all strings containing an odd number of 1s" is a subset of Σ*. It includes 1, 01, 10, 001, 100, … but not ε, 0, 11, 0110, …

🤖 What Is a Finite Automaton? ⭐EXAM

Think of a DFA as a very simple machine — like a vending machine. It has a finite set of states (like the machine's current "mode": waiting for coins, ready to dispense, etc.). It reads one symbol at a time, and each symbol causes it to jump to a new state. At the end, if it is in a special "accept" state, it says "yes, valid!" Otherwise "no".

Formal Definition — DFA

A DFA is a 5-tuple A = (Q, Σ, δ, q₀, F) where:

  • Q — finite set of states
  • Σ — finite input alphabet
  • δ: Q × Σ → Q — transition function: given current state and current symbol, returns the next state
  • q₀ ∈ Q — initial state (where you start)
  • F ⊆ Q — set of final/accepting states

Key constraint: δ is a total function — every state has exactly one outgoing transition per symbol (no gaps, no choices).

🗺️ How to Draw a DFA — Transition Diagrams

DFAs are drawn as directed graphs:

  • Circles = states
  • Arrow into the initial state (no label)
  • Double circle = accepting/final state
  • Labelled arrows = transitions
EXAMPLE: DFA for strings over {0,1} ending in '0' 0 0 → [ s0 ] ←———————— [ s1 ] | | 1 ↓ 1 ↑ [ s1 ] ————————→ * s0 is the initial AND accepting state (double circle) * s1 is a non-accepting state Notation: →[ q ] means initial state (( q )) means accepting state

📋 Transition Tables

The same DFA can be written as a table. Rows = states, columns = symbols, cells = next state. The initial state is marked → and accepting states with *.

🧮 How the DFA Processes a String ⭐EXAM

The extended transition function δ* tells you where you end up after reading an entire string:

Extended Transition Function δ*
  • δ*(q, ε) = q  — reading nothing leaves you in the same state
  • δ*(q, aw) = δ*(δ(q, a), w)  — read the first symbol, move to new state, then continue

The language of DFA A is: L(A) = { w | δ*(q₀, w) ∈ F }

🔍 Step-by-Step Walkthrough — "Strings Ending in 0" ⭐EXAM

Worked Example

DFA with Σ = {0,1}, states = {s0, s1}, start = s0, accepting = {s0}.

Transitions: δ(s0,0)=s0, δ(s0,1)=s1, δ(s1,0)=s0, δ(s1,1)=s1

Trace the string w = "1010":

  1. 1Start in s0. Read '1'. δ(s0, 1) = s1.
  2. 2In s1. Read '0'. δ(s1, 0) = s0.
  3. 3In s0. Read '1'. δ(s0, 1) = s1.
  4. 4In s1. Read '0'. δ(s1, 0) = s0.
  5. 5String finished. Final state = s0 ∈ F. ✓ ACCEPTED (the string ends in 0 ✓).

Trace w = "101":

  1. 1Start s0 → read '1' → s1
  2. 2s1 → read '0' → s0
  3. 3s0 → read '1' → s1. End. s1 ∉ F → ✗ REJECTED (string ends in 1).

🔧 Decision Problems for Regular Languages

  • Membership problem: Given L and w, does w ∈ L? → Run the DFA, check if you end in an accepting state.
  • Emptiness problem: Is L = ∅? → Check if any accepting state is reachable from q₀.
  • Equivalence problem: Do two automata accept the same language? → Use the Equality Game / Table-Filling algorithm.

🔗 Closure Properties of Regular Languages

Closure Properties

If L₁ and L₂ are regular languages, then so are:

  • Union: L₁ ∪ L₂
  • Intersection: L₁ ∩ L₂
  • Complement: Σ* \ L₁
  • Difference: L₁ \ L₂
  • Concatenation: L₁ · L₂ = {xy | x ∈ L₁, y ∈ L₂}
  • Kleene star: L₁* = {ε} ∪ L₁ ∪ L₁L₁ ∪ …
  • Reversal: L₁ᴿ = {wᴿ | w ∈ L₁}

🎮 Equality Game Algorithm (DFA Equivalence) ⭐EXAM

Algorithm
  1. 1Group states: Put all accepting states from both DFAs in one group; all non-accepting states in another.
  2. 2Challenge: A state s in group G challenges s' in G: "On symbol x, I go to group G' — can you?" If s' cannot, split G.
  3. 3Repeat challenges until no split is possible.
  4. 4Verdict: DFAs are equivalent iff their initial states are in the same group at the end.
RL2

Non-Deterministic Finite Automata (NFA)

❓ Why Non-Determinism?

A DFA must have exactly one transition per state per symbol — very rigid. An NFA relaxes this: from any state you can have zero, one, or multiple transitions on the same symbol. The NFA "magically" explores all possibilities at once — it accepts a string if at least one of those explorations leads to an accepting state.

Non-determinism doesn't add power (NFA and DFA recognise exactly the same languages), but it makes it much easier to design automata for complex patterns.

Motivating Example

Language: all strings over {0,1} ending in 01.

The NFA below is compact — it "guesses" when it's in the last two characters:

NFA for strings ending in 01: →[q0] ---0--→ [q1] ---1--→ ((q2)) ↑___________↑ 0,1 (self-loop) q0: keep reading anything q1: just read the '0' before the final '1' q2: accepting — just read '01'

The DFA equivalent would need 4 states (tracking the last two symbols). NFA is simpler to design!

📐 Formal Definition of NFA ⭐EXAM

Definition

An NFA is a 5-tuple A = (Q, Σ, δ, q₀, F) where:

  • Q, Σ, q₀, F — same as DFA
  • δ: Q × Σ → P(Q) — transition function returns a set of states (possibly empty)

The NFA accepts w if at least one computation path reaches an accepting state.

Extended function: δ*(q, ε) = {q} and δ*(q, aw) = ⋃ δ*(p, w) for p ∈ δ(q, a).

Language: L(A) = { w | δ*(q₀, w) ∩ F ≠ ∅ }

🔄 Subset Construction: NFA → DFA ⭐EXAM

Every NFA can be converted to a DFA using the subset construction. Each DFA state corresponds to a set of NFA states — tracking all possible places the NFA could be at once.

Subset Construction Algorithm

Given NFA (Q_N, Σ, δ_N, q₀, F_N), build DFA (Q_D, Σ, δ_D, {q₀}, F_D):

  • Q_D = all subsets of Q_N (the power set P(Q_N)) — potentially 2ⁿ states
  • Start state = {q₀}
  • Accepting states F_D = all subsets containing at least one state from F_N
  • Transitions: δ_D({q₁,…,qₖ}, a) = δ_N(q₁,a) ∪ … ∪ δ_N(qₖ,a)

In practice, only the reachable subsets matter — often far fewer than 2ⁿ.

Step-by-Step Walkthrough — NFA to DFA Subset Construction

Given NFA with Q = {p, q, r}, Σ = {0, 1}, start = p, accepting = {r}:

State01
→ p{p,q}{p}
q{r}{r}
* r∅∅
  1. 1Start: DFA start state = {p}.
  2. 2δ_D({p}, 0) = δ_N(p,0) = {p,q} → new DFA state.
  3. 3δ_D({p}, 1) = δ_N(p,1) = {p} → already known.
  4. 4δ_D({p,q}, 0) = δ_N(p,0) ∪ δ_N(q,0) = {p,q} ∪ {r} = {p,q,r} → new state.
  5. 5δ_D({p,q}, 1) = δ_N(p,1) ∪ δ_N(q,1) = {p} ∪ {r} = {p,r} → new state.
  6. 6Continue expanding all new states until no new states appear.
  7. 7Accepting states: Any DFA state containing 'r' is accepting, i.e. {p,q,r}, {p,r}, {r}.

ε-NFAs (NFA with Epsilon Moves)

An ε-NFA allows free/silent transitions — the automaton can jump from one state to another without consuming any input symbol. This makes composition of automata (e.g., union) trivially easy.

ε-Closure

ECLOSE(q) = the set of all states reachable from q by following zero or more ε-transitions.

Formally: q ∈ ECLOSE(q), and if p ∈ ECLOSE(q) and r ∈ δ(p,ε), then r ∈ ECLOSE(q).

Worked Example — ε-Closure and ε-NFA to DFA

ε-NFA over {a, b, c}:

Stateεabc
→ p∅{p}{q}{r}
q{p}{q}{r}∅
* r{q}{r}∅{p}
  1. 1Compute ε-closures: ECLOSE(p)={p}, ECLOSE(q)={q,p} (q has ε to p), ECLOSE(r)={r,q,p} (r→q→p via ε).
  2. 2DFA start = ECLOSE(p) = {p}.
  3. 3δ_D({p}, a) = ECLOSE(δ(p,a)) = ECLOSE({p}) = {p}.
  4. 4δ_D({p}, b) = ECLOSE(δ(p,b)) = ECLOSE({q}) = {p,q}.
  5. 5δ_D({p}, c) = ECLOSE(δ(p,c)) = ECLOSE({r}) = {p,q,r}.
  6. 6{p,q,r} contains r (accepting), so it is an accepting DFA state. Continue until stable.

🗺️ The Big Picture — Equivalences

DFA ←——subset construction—— NFA ←——trivial embedding—— ε-NFA | | ↓ ↓ Regular Expressions ←————————————————————————————————→ same class! All four formalisms describe exactly the SAME class: Regular Languages.

Key theorem: L is accepted by a DFA ⟺ L is accepted by an NFA ⟺ L is accepted by an ε-NFA ⟺ L is described by a regular expression.

RL3

Regular Expressions

📝 What Is a Regular Expression?

A regular expression (regex) is an algebraic formula that describes a language. Instead of drawing a diagram, you write a compact mathematical expression. For example, (0+1)*1 describes all binary strings ending in 1.

Regular expressions are used everywhere: text editors, compilers (to define keywords), search tools like grep.

🔧 Syntax of Regular Expressions ⭐EXAM

Grammar of RegEx

A regular expression E is one of:

ExpressionLanguage L(E)Meaning
∅∅Empty language (no strings)
ε{ε}Only the empty string
a (a ∈ Σ){a}Single character
E₁ + E₂L(E₁) ∪ L(E₂)Union (either or both)
E₁ · E₂ or E₁E₂L(E₁) · L(E₂)Concatenation
E*L(E)*Kleene star: zero or more repetitions

Precedence: * binds tightest, then ·, then +. So ab*+c = (a(b*))+c.

Reading Regular Expressions — Examples
RegExLanguage
(a+b)*All strings over {a,b} (including ε)
(a+b)*a(a+b)(a+b)All strings where the 3rd-from-last char is 'a'
(0+1)*01(0+1)*All binary strings containing "01" as substring
alpha(alpha+digit)*Identifiers: start with letter, then any alphanumeric

🏗️ Thompson's Construction: RegEx → ε-NFA ⭐EXAM

Every regular expression can be mechanically converted into an ε-NFA, and then to a DFA. The construction is structural — it mirrors the structure of the expression.

Thompson's Construction Rules
Base cases: ∅: → [i] (no accepting state reachable) ε: → [i] --ε-→ ((f)) a: → [i] --a-→ ((f)) Inductive cases — E₁ + E₂ (union): Create new start i and new final f. i --ε-→ start(E₁) --...E₁...-→ final(E₁) --ε-→ f i --ε-→ start(E₂) --...E₂...-→ final(E₂) --ε-→ f E₁ · E₂ (concatenation): Run E₁, then ε-connect its final state to E₂'s start. → [i] --E₁--→ [m] --ε-→ [n] --E₂--→ ((f)) E₁* (Kleene star): Create new start i, new final f. i --ε-→ start(E₁) --E₁--→ final(E₁) --ε-→ f final(E₁) --ε-→ start(E₁) (back edge for repetition) i --ε-→ f (ε path for zero repetitions)
Worked Example — Build ε-NFA for RegEx "1 0*"
  1. 1Build ε-NFA for 1: → [i₁] --1--→ ((f₁))
  2. 2Build ε-NFA for 0: → [i₀] --0--→ ((f₀))
  3. 3Apply Kleene star to 0: new start s, new final t; add ε: s→i₀, f₀→i₀ (loop), f₀→t, s→t.
  4. 4Concatenate 1 with 0*: connect f₁ with ε to s (start of 0*).
  5. 5Final result: i₁ --1-→ f₁ --ε-→ s --ε-→ i₀ --0-→ f₀, with f₀ looping back to i₀ and f₀ --ε-→ t (accepting).
→[i₁] --1-→ [f₁] --ε-→ [s] --ε-→ [i₀] --0-→ [f₀] | | ↑ ε←←←←←←←←←←←←←←←←ε | ↓ ε___↘ (loop) ((t))←←←←←ε←←←←←←←←←

🔄 State Elimination: DFA/NFA → RegEx ⭐EXAM

Going the other way — converting a DFA to a regex — uses state elimination. Remove states one by one, labelling edges with regular expressions to capture the eliminated state's paths.

State Elimination Rule
Before (state q to eliminate): [A] --R--→ [q] --S--→ [B] ↑___T____↓ (self-loop on q) [A] --U--→ [B] After eliminating q: [A] --R·T*·S + U--→ [B] (All paths that went through q are now captured in one regex label)
Worked Example — NFA to RegEx (State Elimination)

NFA with states {1, 2, 3}, start=1, accepting={3}, transitions: 1→2 on 0, 1→2 on 1, 2→2 on 0, 2→3 on 1, 3→1 on 0.

  1. 1For each final state (here just 3), eliminate all intermediate states to find a regex from start to that state.
  2. 2Eliminate state 2: path 1→2→3 becomes 1→3 labelled (0+1)·0*·1. Path via 3→1→2→3 contributes to a loop.
  3. 3Eliminate state 1 (keeping just start and final 3): fold the loop through 3→1→2→3 as a Kleene star.
  4. 4Result: regex = (0+1)* (start accepts ε via direct path) + (0+1)*·1·(0(0+1)*1)*

In exams, just apply the pattern rules mechanically. The key pattern to memorise:

Two-state NFA [i] --R-→ ((f)): regex = R Three-state with self-loop: [i] --A-→ [q] --B-→ ((f)) ↑___C___↓ regex = A·C*·B

⚖️ Laws for Regular Expressions

Key Laws (hold for all E, F, G)
  • E + F = F + E (union is commutative)
  • (E + F) + G = E + (F + G) (union is associative)
  • E · ε = ε · E = E (ε is concatenation identity)
  • E · (F · G) = (E · F) · G (concatenation is associative)
  • E · (F + G) = E·F + E·G (distributivity)
  • (E*)* = E*
  • ∅* = ε
  • NOTE: E·F ≠ F·E in general — concatenation is NOT commutative!

Minimization, Myhill-Nerode, and Non-Regularity

🗜️ DFA Minimization — Why?

The same language may be accepted by DFAs with very different numbers of states. Fewer states = simpler, faster. We want the smallest possible DFA. The block refinement algorithm finds it.

📦 Block Refinement (Table-Filling / Partition Refinement) ⭐EXAM

Algorithm
  1. 1Initial partition Π: One block = all accepting states; one block = all non-accepting states.
  2. 2Refine: Find two states p, q in the same block but on symbol a they transition to different blocks. If so, split their block.
  3. 3Repeat until no more splits possible.
  4. 4Minimal DFA: Each block becomes one state. Transitions follow accordingly.

States in the same block at the end are indistinguishable — merging them doesn't change the language.

Worked Example — DFA Minimization

DFA: States {0, 1, 2}, Σ={a,b}, start=0, accepting={1}. Transitions:

Stateab
→ 012
* 112
222
  1. 1Initial Π = {{1}, {0,2}} (accepting vs non-accepting).
  2. 2Check {0,2}: on 'a', 0→1 (block {1}), 2→2 (block {0,2}). Different blocks → split! Π = {{1},{0},{2}}.
  3. 3Check {0}: singleton, no split. Check {2}: singleton. Check {1}: singleton. No more splits.
  4. 4All blocks are singletons → this DFA was already minimal. (But note: 0 and 1 have identical transitions — they should be checked. On 'a': both → {1} block; on 'b': both → {2} block → they ARE distinguishable since 0 is not accepting but 1 is. Correct split.)

The minimised DFA has 3 states as derived from the partition.

🧮 Myhill-Nerode Equivalence ⭐EXAM

This is the deep theoretical foundation. It gives us both the minimal DFA and a criterion for whether a language is regular at all.

Myhill-Nerode Equivalence ~_L

Given language L ⊆ Σ*, two strings x, y are L-equivalent (written x ~_L y) if:

for all z ∈ Σ*: xz ∈ L ⟺ yz ∈ L

"x and y are equivalent if every extension behaves the same with respect to L membership."

The equivalence classes of ~_L are the states of the minimal DFA.

Myhill-Nerode Theorem

L is regular ⟺ ~_L has finitely many equivalence classes.

The number of equivalence classes = the number of states in the minimal DFA.

Worked Example — Myhill-Nerode Equivalence Classes

Language: L = {1ⁿ | n ≥ 1} (non-empty strings of all 1s), Σ = {0,1}.

  1. 1[ε]_L: Which extensions z make εz ∈ L? Need z ∈ {1,11,111,…}. So [ε]_L = {ε}.
  2. 2[1]_L: 1z ∈ L iff z ∈ {ε, 1, 11,…}. This is the same condition as "extend ε by z": [1]_L = L itself.
  3. 3[0]_L: 0z ∈ L iff z makes "0z" all-1s — impossible. So [0]_L = Σ* \ L.
  4. 4Three classes: [ε], [1]=L, [0]=Σ*\L. So minimal DFA has 3 states.
Minimal DFA for L = {1ⁿ | n ≥ 1}: →[ε] --1-→ ((L)) | | 0 1 (self) ↓ ↓ [0*] ←-0-- [0*] ↑___0,1____↓ (sink)

🚫 Proving a Language Is NOT Regular ⭐EXAM

Three approaches:

Approach 1: Infinite Myhill-Nerode Classes

Example

Show {aⁿbⁿ | n ≥ 0} is not regular.

  • Consider strings aⁱ and aʲ for 0 ≤ i < j.
  • Choose z = bⁱ. Then aⁱbⁱ ∈ L but aʲbⁱ ∉ L.
  • So aⁱ ≁_L aʲ — each aⁱ is in a different equivalence class.
  • Infinitely many classes → not regular. ∎

Approach 2: Closure Properties Reduction

Example

Show L = {aⁿbᵐ | n ≥ m} is not regular by contradiction:

  • Suppose L is regular.
  • L' = {aⁿbᵐ | n ≤ m} is also regular (by symmetry/closure).
  • L ∩ L' = {aⁿbⁿ | n ≥ 0} must then be regular.
  • But {aⁿbⁿ} is not regular (shown above). Contradiction!

Approach 3: The Pumping Lemma ⭐EXAM

Pumping Lemma

If L is regular, then there exists a "pumping length" n such that for every w ∈ L with |w| ≥ n, we can write w = xyz where:

  • |xy| ≤ n
  • |y| ≥ 1 (y is non-empty)
  • xy^k·z ∈ L for all k ≥ 0

Intuition: Long strings in a regular language must contain a "loop" that can be repeated (pumped) any number of times.

Worked Example — Pumping Lemma Proof

Claim: L = {aⁿbⁿ | n ≥ 0} is not regular.

  1. 1Assume for contradiction that L is regular. Let n be the pumping length.
  2. 2Choose w = aⁿbⁿ ∈ L with |w| = 2n ≥ n.
  3. 3By the lemma, w = xyz with |xy| ≤ n, |y| ≥ 1. Since |xy| ≤ n, xy consists entirely of a's. So y = aᵏ for some k ≥ 1.
  4. 4Pump: Take k = 2. Then xy²z = aⁿ⁺ᵏbⁿ. This has more a's than b's, so it's NOT in L.
  5. 5Contradiction with the lemma (xy²z must be in L). Therefore L is not regular. ∎
Common Pumping Lemma Mistakes
  • Don't confuse direction: the pumping lemma says "if regular, then pumpable." To prove non-regular, you assume regularity and derive contradiction.
  • You must show the contradiction works for all possible splittings xyz — you pick w, but the adversary picks x, y, z (within the constraints).
  • The pumping lemma cannot prove a language IS regular.

Beyond Regular Languages — Pushdown Automata, Turing Machines, Undecidability

🌊 The Language Hierarchy

Nested hierarchy of formal languages: ┌──────────────────────────────────────────────────────┐ │ All languages (uncountably many) │ │ ┌────────────────────────────────────────────────┐ │ │ │ Decidable/Recognisable (Turing machines) │ │ │ │ ┌──────────────────────────────────────────┐ │ │ │ │ │ Context-Free Languages (PDAs, CFGs) │ │ │ │ │ │ ┌────────────────────────────────────┐ │ │ │ │ │ │ │ Regular Languages │ │ │ │ │ │ │ │ (DFA / NFA / ε-NFA / RegEx) │ │ │ │ │ │ │ └────────────────────────────────────┘ │ │ │ │ │ └──────────────────────────────────────────┘ │ │ │ └────────────────────────────────────────────────┘ │ └──────────────────────────────────────────────────────┘ Example: Regular: (a+b)* aⁿ for n≥0 Context-Free: aⁿbⁿ balanced brackets Decidable: aⁿbⁿcⁿ (needs Turing machine) Undecidable: Halting problem (no algorithm exists)

📚 Pushdown Automata (PDA) ⭐EXAM

A pushdown automaton is a DFA with an unbounded stack. The stack gives it memory to count — enough to recognise context-free languages like {aⁿbⁿ}.

Why only a stack and not more? Because the richer the model, the harder the problems become (equivalence for PDAs is already undecidable).

Formal Definition — PDA

A PDA is a 7-tuple A = (Q, Σ, Γ, q₀, Z₀, δ, F) where:

  • Q — finite states
  • Σ — input alphabet
  • Γ — stack alphabet
  • q₀ — initial state
  • Z₀ ∈ Γ — initial stack bottom symbol
  • δ ⊆ Q × (Σ∪{ε}) × Γ × Q × Γ* — transition relation: (state, input, top-of-stack) → (new state, string to push)
  • F ⊆ Q — accepting states

A transition (q, a, Z, q', γ) means: in state q, read input a, pop Z from stack, go to q', push γ onto stack.

Worked Example — PDA for aⁿbⁿ

Language: {aⁿbⁿ | n ≥ 0} — push an 'a' symbol for each 'a' read, pop one for each 'b'.

PDA: States {0, 1, 2, 3}, Σ={a,b}, Γ={Z, Z₀}, start=0, accepting={3} Transitions (notation: "input, top-of-stack / push"): 0 → 1: a, Z₀ / ZZ₀ (read first 'a', push Z) 1 → 1: a, Z / ZZ (read more 'a's, push Z for each) 1 → 2: b, Z / ε (start reading 'b's, pop one Z) 2 → 2: b, Z / ε (continue popping) 2 → 3: ε, Z₀ / Z₀ (stack back to bottom — accept!)

Trace w = "aabb":

  1. 1Config (0, aabb, Z₀) → read 'a', push Z: (1, abb, ZZ₀)
  2. 2Config (1, abb, ZZ₀) → read 'a', push Z: (1, bb, ZZZ₀)
  3. 3Config (1, bb, ZZZ₀) → read 'b', pop Z: (2, b, ZZ₀)
  4. 4Config (2, b, ZZ₀) → read 'b', pop Z: (2, ε, Z₀)
  5. 5Config (2, ε, Z₀) → ε-move to (3, ε, Z₀). State 3 ∈ F → ACCEPTED.

🖥️ Turing Machines and Decidability

A Turing Machine (TM) is the most powerful theoretical computational model. It has an infinite tape (not just a stack) and can read/write freely. Every "reasonable" computation model has the same power as a TM — this is the Church-Turing hypothesis.

Formal Definition — Turing Machine

A TM is a 7-tuple M = (Q, Σ, Γ, B, q₀, qf, δ) where:

  • Q — finite states; Σ — input alphabet; Γ ⊃ Σ — tape alphabet
  • B ∈ Γ\Σ — blank symbol; q₀ — initial state; qf — final state
  • δ ⊆ (Q\{qf}) × Γ × Q × Γ × {R,L,N} — transition: (state, read) → (new state, write, move head)

L is decidable iff there is a TM M with L(M) = L (M always halts).

♾️ Undecidability — Some Problems Have No Algorithm ⭐EXAM

There are more languages than Turing machines (languages are uncountably many; TMs are countably many). Therefore some languages must be undecidable.

The Halting Problem — Classic Undecidable Problem

H = { (M, w) | TM M terminates on input w }

Theorem: H is undecidable. There is no algorithm that can determine whether an arbitrary program terminates on a given input.

Proof sketch (diagonalisation): Assume M_H decides H. Construct M_D that asks M_H whether M_i accepts w_i — then does the opposite. This leads to a contradiction (M_D cannot accept or reject D without contradiction). ∎

Rice's Theorem

Any non-trivial property of the language accepted by a TM is undecidable.

Consequences: No algorithm can decide whether a program: always terminates, returns only prime numbers, sorts correctly, or has any specific functional property.

Note: "AI will always make mistakes when asked to solve such problems" — slides verbatim.

Subroutine Technique for Proving Undecidability

To show language L is undecidable:

  1. Assume L is decidable (there is a TM M_L).
  2. Use M_L as a subroutine to decide a known undecidable problem (like H or D).
  3. Contradiction — M_L cannot exist.

🔮 Countability and Why Undecidable Problems Exist

  • The set of all Turing machines is countably infinite (they can be encoded as finite binary strings).
  • The set of all languages (P(Σ*)) is uncountably infinite (by Cantor's diagonalisation argument).
  • Therefore: most languages are not decidable by any TM.
Summary

⭐ Key Takeaways

These are the 5 most critical things to understand for the exam:

1
DFA, NFA, ε-NFA, and Regular Expressions are all equivalent. They describe exactly the class of regular languages. You must be able to convert between them: RegEx → ε-NFA (Thompson), ε-NFA → DFA (subset construction + ε-closure), DFA → RegEx (state elimination). Exam questions very often ask for one or more of these conversions.
2
Running a DFA on a string means tracing δ* step by step. Start at q₀, consume one symbol at a time to compute the next state, accept iff the final state is in F. You will be asked to do this in the exam — show every step clearly.
3
The Pumping Lemma and Myhill-Nerode are the tools for proving non-regularity. For the pumping lemma: assume the language is regular, pick a long word, show it cannot be pumped into the language. For Myhill-Nerode: find infinitely many strings that are pairwise distinguishable (go to different equivalence classes). Both approaches appear in every year's exam paper.
4
DFA minimization uses block refinement: start with {F, Q\F}, refine until stable. States in the same block are indistinguishable and can be merged. The minimal DFA corresponds exactly to the Myhill-Nerode equivalence classes. Exam questions often give you a larger DFA and ask for the minimal one.
5
Regular languages sit at the bottom of the language hierarchy. Languages requiring counting (like {aⁿbⁿ}) are context-free but not regular — they need PDAs. Problems like the Halting Problem are not even decidable. In the exam, you may need to identify which class a language belongs to and explain why.

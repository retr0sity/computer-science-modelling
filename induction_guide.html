§ 01

Mathematical Induction (on natural numbers)

What it is and why it works

Mathematical induction is a proof technique for statements of the form "P(n) holds for every natural number n ≥ n₀". Think of it as an infinite row of dominoes: prove the first one falls (base case), and prove that whenever one falls the next one falls too (inductive step). Together they guarantee every domino eventually falls.

Formally this rests on the well-ordering principle: every non-empty set of natural numbers has a smallest element. If P failed for some n, there would be a smallest failing n — and the inductive step would produce a contradiction.

Standard proof structure ⭐ exam format

Template — copy this structure into every integer induction proof
1. State P(n)
Let P(n) be the statement: [write out the property in terms of n explicitly].
2. Base case
We prove P(n₀). Verify the statement directly for n = n₀ by substitution or direct computation.
3. Ind. hypothesis
Assume P(n) holds for some arbitrary n ≥ n₀. [This is the I.H. — write it out as an equation or statement.]
4. Inductive step
Prove P(n+1) using the I.H. Every algebraic step should be labelled: [by I.H.], [by definition], [by algebra].
5. Conclusion
By induction, P(n) holds for all n ≥ n₀. □
⚠ Common pitfalls
  • Forgetting to state P(n). Examiners need to see exactly what is being proved. Don't just write "by induction" — spell out P(n).
  • Wrong base case. If the claim starts at n = 1 but you check n = 0, the whole proof collapses.
  • Circular reasoning. Don't assume P(n+1) and derive P(n+1). You must start from P(n) and arrive at P(n+1).
  • Unjustified algebra. Every equals sign in the inductive step needs a reason in brackets.
§ 02

Strong Induction

When to use it

In regular induction you assume P(n) to prove P(n+1). In strong induction you assume all of P(0), P(1), …, P(n) to prove P(n+1). This is necessary when the proof of P(n+1) depends on a case P(k) where k < n — for example when a problem breaks into two smaller sub-problems of unknown size.

Use regular induction when…

  • P(n+1) depends only on P(n)
  • Typical for sum/product formulas, simple counting

Use strong induction when…

  • P(n+1) needs P(k) for some k < n (not just k = n)
  • Recursive definitions split into two pieces of unknown size
  • Fibonacci-style arguments

Proof structure

Template — strong induction
1. State P(n)
Let P(n) be: [property].
2. Base case(s)
Verify P(n₀) and any other small cases needed before the step applies.
3. Strong I.H.
Assume P(i) holds for all i with n₀ ≤ i ≤ n.
4. Inductive step
Prove P(n+1). You may now invoke P(i) for any i ≤ n — not just P(n).
5. Conclusion
By strong induction, P(n) holds for all n ≥ n₀. □
§ 03

Structural Induction

What it is

Structural induction is induction over recursively defined structures rather than over the natural numbers. Instead of going from n to n+1, you go from sub-structures to the structures built from them. To prove a property holds for every element of a recursively defined set, prove it for all base cases, then prove it is preserved by each recursive construction rule.

How structural and integer induction relate

Structural induction is a generalisation of integer induction. When you do induction on the length of a string you are really doing integer induction on |w|. Structural induction makes this explicit: the "smaller" pieces are the sub-structures given directly by the grammar rules.

Standard proof structure ⭐ exam format

Template — structural induction over a grammar
1. State P
We prove by structural induction that P(x) holds for every x in the set S.
2. Base cases
For each base production rule, verify P directly. One base case per base production.
3. Inductive cases
For each recursive rule that builds x from smaller pieces x₁, x₂, …: assume P(x₁), P(x₂), … hold (I.H.), then prove P(x) for the newly built element. One case per recursive production.
4. Conclusion
By structural induction, P(x) holds for all x ∈ S. □

Where it appears in this course

strings / words parse trees regular expressions context-free grammars derivation sequences S ⇒* w GCL programs

For a CFG, base cases match productions that derive terminal words directly (e.g. S → cd). Inductive cases match productions that use S recursively (e.g. S → cSd, S → SS). There is always one case per production rule.

§ 04

Induction on String Length

Why strings need their own section

Many results about automata are proved by induction on |w|. This is technically integer induction, but the argument looks different because you split words by stripping a character, not by doing arithmetic.

Standard split conventions
  • Strip the last character: write w = w'a where a ∈ Σ and |w'| = |w| − 1. Most common for DFA/NFA transition function proofs.
  • Strip the first character: write w = aw'. Sometimes used for left-to-right reading arguments.
  • Split in two: write w = w₁w₂ where |w₁|, |w₂| < |w| (requires strong induction).

Proof structure for induction on |w| ⭐ exam format

Template — induction on string length
1. State P(w)
We prove by induction on |w| that P(w) holds for every w ∈ Σ*.
2. Base case w = ε
Verify P(ε) directly using δ*(q, ε) = q.
3. Ind. hypothesis
Assume P(w') holds for all words w' with |w'| = n (or: for all w' shorter than w).
4. Inductive step
Let w = w'a where a ∈ Σ and |w'| = n. By I.H., P(w') holds. Use the recursive definition of δ* to derive P(w).
5. Conclusion
By induction on |w|, P(w) holds for all w ∈ Σ*. □

The recursive definition of δ* — memorise this ⭐

δ*(q, ε) = q δ*(q, w'a) = δ(δ*(q, w'), a) [for any w' ∈ Σ*, a ∈ Σ]

This recursive structure tells you exactly what the base case and inductive step must look like. The base case is always w = ε giving δ*(q, ε) = q. The inductive step always strips the last character a and uses the definition twice.

§ 05–09

Worked Examples

Every step is labelled explicitly, as it should be in an exam answer.

EXAMPLE 1 Integer induction — prove the sum formula

Claim: For all n ≥ 0,   0 + 1 + 2 + ⋯ + n = n(n+1)/2.

  • Step 1
    State P(n)
    Let P(n) be: the sum 0 + 1 + ⋯ + n equals n(n+1)/2. We want to show P(n) for all n ≥ 0.
  • Step 2
    Base case
    For n = 0: the sum is 0, and 0(0+1)/2 = 0. ✓ So P(0) holds.
  • Step 3
    Ind. hypothesis
    Assume P(n) holds for some arbitrary n ≥ 0:
    0 + 1 + 2 + ⋯ + n = n(n+1)/2 [I.H.]
  • Step 4
    Ind. step
    We prove P(n+1), i.e. 0 + 1 + ⋯ + n + (n+1) = (n+1)(n+2)/2.
    0 + 1 + ⋯ + n + (n+1) = [0 + 1 + ⋯ + n] + (n+1) [regrouping] = n(n+1)/2 + (n+1) [by I.H.] = (n+1) · [n/2 + 1] [factoring] = (n+1)(n+2)/2 [algebra] ✓
  • Step 5
    Conclusion
    By induction, P(n) holds for all n ≥ 0. □
EXAMPLE 2 Strong induction — every integer ≥ 8 is a sum of 3s and 5s

Claim: For all n ≥ 8, n = 3a + 5b for some a, b ≥ 0.

  • Step 1
    State P(n)
    Let P(n) be: n = 3a + 5b for some a, b ∈ ℕ₀.
  • Step 2
    Base cases
    Three base cases are needed (because the step will use n−3):
    • P(8): 8 = 3 + 5 ✓
    • P(9): 9 = 3 + 3 + 3 ✓
    • P(10): 10 = 5 + 5 ✓
  • Step 3
    Strong I.H.
    Assume P(i) holds for all i with 8 ≤ i ≤ n, where n ≥ 10.
  • Step 4
    Ind. step
    We prove P(n+1). Since n ≥ 10, (n+1) − 3 = n − 2 ≥ 8. By I.H. applied to n − 2 (which satisfies 8 ≤ n−2 ≤ n), we have n−2 = 3a + 5b. Therefore n+1 = (n−2) + 3 = 3(a+1) + 5b. ✓

    Note: strong induction was essential — P(n+1) needed P(n−2), not P(n).
  • Step 5
    Conclusion
    By strong induction, every n ≥ 8 can be written as 3a + 5b. □
EXAMPLE 3 Structural induction on a grammar ⭐ 2024 exam style

Grammar G:  S → cd | cSd | SS

Claim: For every word w derivable from S, |w|d ≥ |w|c, where |w|x denotes the number of occurrences of x.

This matches the style of Question 1d from May 2024.

  • Step 1
    State P
    We prove by structural induction on the derivation from G that for every w with S ⇒* w (all terminals), |w|d ≥ |w|c.
  • Step 2
    Base case
    S → cd
    w = cd. Then |w|d = 1 and |w|c = 1, so |w|d ≥ |w|c. ✓
  • Step 3
    Ind. case
    S → cSd
    Suppose S ⇒* w₀ and P(w₀) holds, i.e. |w₀|d ≥ |w₀|c.
    Then w = cw₀d, so |w|c = |w₀|c + 1 and |w|d = |w₀|d + 1.
    By I.H.: |w|d = |w₀|d + 1 ≥ |w₀|c + 1 = |w|c. ✓
  • Step 4
    Ind. case
    S → SS
    Suppose S ⇒* w₁ and S ⇒* w₂ with P(w₁) and P(w₂) both holding.
    Then w = w₁w₂, so |w|d = |w₁|d + |w₂|d ≥ |w₁|c + |w₂|c = |w|c. ✓
  • Step 5
    Conclusion
    By structural induction, |w|d ≥ |w|c for every word w derivable from S. □
EXAMPLE 4 Induction on |w| — the concatenation lemma for δ* ⭐ frequently tested

Claim: For any DFA (Q, Σ, δ, q₀, F), any q ∈ Q, and any x, y ∈ Σ*:

δ*(q, xy) = δ*(δ*(q, x), y)

Plain English: reading x then y from q gives the same result as reading x from q, then reading y from wherever you land.

  • Step 1
    State P(y)
    We prove by induction on |y| that for all q ∈ Q and all x ∈ Σ*: δ*(q, xy) = δ*(δ*(q, x), y). We fix x and induct on y.
  • Step 2
    Base case y = ε
    δ*(q, xε) = δ*(q, x) [since xε = x] δ*(δ*(q, x), ε) = δ*(q, x) [by def: δ*(r, ε) = r]
    Both sides equal δ*(q, x). P(ε) holds. ✓
  • Step 3
    Ind. hypothesis
    Assume δ*(q, xy) = δ*(δ*(q, x), y) for all q ∈ Q, all x ∈ Σ*.  [I.H. for word y]
  • Step 4
    Ind. step y' = ya
    Let y' = ya where a ∈ Σ.
    δ*(q, xy') = δ*(q, x(ya)) = δ*(q, (xy)a) [concatenation is associative] = δ(δ*(q, xy), a) [def of δ*: strip last char] = δ(δ*(δ*(q, x), y), a) [by I.H.] = δ*(δ*(q, x), ya) [def of δ*: reversed] = δ*(δ*(q, x), y') ✓
  • Step 5
    Conclusion
    By induction on |y|, δ*(q, xy) = δ*(δ*(q, x), y) for all q, x, y. □
Why this lemma matters

This is the core tool for almost every induction proof about DFAs. Once proved, you can split any word at any position and reason about the two halves independently — which is exactly what Example 5 below does.

EXAMPLE 5 May 2023 exam — the dup function and DFA A₂ ⭐ appeared May 2023

This is drawn directly from Exercise 8 of the May 2023 exam (5% of total marks). It is the most representative exam-style induction question in this course.

Setup. The function dup : Σ* → Σ* doubles every character:

dup(ε) = ε dup(w'a) = dup(w') aa [for w' ∈ Σ*, a ∈ Σ]

Given a DFA A = (Q, Σ, δ, q₀, F), construct DFA A₂ = (Q₂, Σ, δ₂, q₀, F) where intuitively A₂ reads each character twice before moving on. The transition function is:

δ₂(q, a) = q_a [if q ∈ Q — first occurrence, go to intermediate state] δ₂(q_a, a) = δ(q, a) [if q is of the form q'_a — second occurrence, complete the move] δ₂(_, _) = q_e [otherwise — error]

Claim to prove: For all q ∈ Q and w ∈ Σ*:

δ*(q, w) = δ₂*(q, dup(w))
  • Step 1
    State P(w)
    We prove by induction on |w| that for all q ∈ Q:
    P(w): δ*(q, w) = δ₂*(q, dup(w)).
  • Step 2
    Base case w = ε
    δ*(q, ε) = q [def of δ*] δ₂*(q, ε) = q [def of δ₂*] δ₂*(q, dup(ε)) = δ₂*(q, ε) = q [since dup(ε) = ε]
    Both sides equal q. P(ε) holds. ✓
  • Step 3
    Ind. hypothesis
    Assume P(w') holds for all q ∈ Q, i.e. δ*(q, w') = δ₂*(q, dup(w')) for all q ∈ Q.  [I.H.]
  • Step 4
    Ind. step w = w'a
    Let w = w'a where a ∈ Σ. We show δ*(q, w'a) = δ₂*(q, dup(w'a)).
    Left side: Let q'' = δ*(q, w') and q' = δ(q'', a). Then δ*(q, w'a) = δ(δ*(q, w'), a) = δ(q'', a) = q'. [def of δ*] Right side: dup(w'a) = dup(w') aa [def of dup] By I.H.: δ₂*(q, dup(w')) = δ*(q, w') = q'' By def δ₂: δ₂(q'', a) = q''_a [first occurrence] By def δ₂: δ₂(q''_a, a) = δ(q'', a) = q' [second occurrence] So: δ₂*(q, dup(w'a)) = δ₂*(q, dup(w') aa) = q' Therefore δ*(q, w'a) = q' = δ₂*(q, dup(w'a)). ✓
  • Step 5
    Conclusion
    By induction on |w|, δ*(q, w) = δ₂*(q, dup(w)) for all q ∈ Q, w ∈ Σ*.
    Setting q = q₀: δ*(q₀, w) ∈ F iff δ₂*(q₀, dup(w)) ∈ F. So A₂ accepts dup(w) iff A accepts w. □
§ 10

Exam Writing Tips

How to identify which induction to use

Clues in the problem wording

  • "…for all n ≥ 0…" → integer induction
  • "…by induction on |w|…" → string length
  • "…structural induction on the derivation…" → grammar
  • "…every word generated by…" → structural on grammar
  • Recursively defined function (like dup) → induct on the same recursion

Decision checklist

  • Object is a natural number or has a size? → integer induction
  • n+1 case needs more than just n? → strong induction
  • Object defined by grammar / recursive rule? → structural
  • Word/string with function over it? → induction on |w|

Writing clearly in an exam ⭐

  • Always name the I.H. Write "By I.H." explicitly when you use it. Examiners need to see where it is applied.
  • Label all three parts. Write "Induction base:" and "Induction step:" as explicit headings — don't just start computing.
  • State P(n) or P(w) first, even if it seems obvious. This is worth marks by itself.
  • Justify every = sign. Bracket each step: [by def of δ*], [by I.H.], [by construction of A₂], [algebra].
  • End with a conclusion sentence. "By induction on |w|, P holds for all w ∈ Σ*. □"
  • Match base case to step. If your step splits w = w'a, your base case must handle ε.

Common exam phrasings to recognise

"Prove by induction on the length of w…" → induction on |w|; base case w = ε; step w = w'a "By structural induction on the syntax of expression E…" → base cases = atomic expressions; inductive cases = each operator "Complete the proof below by filling in the blank lines…" → use the exact template the exam provides; do not invent a different structure "We first show by induction on the structure of words that…" → this is the 2023 exam phrasing; it means induction on |w|

Time management

  • Induction proofs in 02141 are typically worth 5% of the exam (e.g. Exercise 8 in 2023). Budget 8–12 minutes maximum.
  • If stuck on the inductive step, write the base case and the I.H. statement — these alone may earn partial credit.
  • The inductive step is almost always: unfold the recursion definition, apply I.H., refold. Recognise this pattern and it becomes mechanical.

// KEY TAKEAWAYS

1
Always write the three-part structure explicitly. State P(n) / P(w), prove the base case, state the I.H., prove the inductive step. Exams often give you the skeleton — fill each part carefully and label it. ⭐
2
Induction on |w| mirrors the recursive definition of δ*. Base case is always w = ε with δ*(q, ε) = q. The step is always w = w'a — apply δ* to w' then δ to the last character. This pattern appeared in 2023 and is the most likely type to reappear. ⭐
3
Structural induction has one case per production rule. List all productions. Handle base rules as base cases; for each recursive rule, assume P for the sub-derivations and prove P for the result. The 2021 and 2024 exams both used this pattern. ⭐
4
Use strong induction when the step reaches back more than one level. The signal is a recursion that splits into two sub-parts of unknown size (like dup(w'a) = dup(w')aa). Three or more base cases are often needed.
5
Every equality needs a justification in brackets. [by I.H.], [by def of δ*], [by construction of A₂], [algebra] — these show the examiner you know why each step is valid and are not just asserting the conclusion. ⭐
